SafeLibrary Pages

Usage Guide

This guide explains how to actually use HouseLearning Pages after setup — from creating sites to uploading files and managing deployments. No theory, just how it works.

1. Creating a New Site

After logging in to HouseLearning Pages, users can create a site directly from the dashboard.

  • Click Create New Site
  • Choose a site name (used in the URL)
  • The platform provisions storage and deployment automatically

Each account has a site limit to prevent abuse and keep hosting free.

2. Uploading Files

HouseLearning Pages only supports static files. That’s intentional. 

✔ index.html
✔ styles.css
✔ app.js
✔ images (.png, .jpg, .gif)
✖ server-side code
✖ databases
✖ executables

Files are validated before upload and stored securely before deployment.

3. Deploying Your Site

Deployment is automatic. There is no “build” step.

  1. Upload files
  2. Click deploy (or auto-deploy triggers)
  3. A GitHub Action commits files
  4. GitHub Pages serves the site

Typical deploy time: 10–30 seconds.

4. Accessing Your Site

Every site is published under a predictable URL:

https://pages.houselearning.org/<username>/<site-name>/

Custom domains may be supported later — Pages prioritizes simplicity and safety first.

5. Updating & Deleting Files

Updating a site is just re-uploading files.

  • Replace files to update content
  • Deleted files are removed from deployment
  • All changes redeploy automatically
