Getting Started

How to Set Up Your Page

This guide walks you through setting up your first HouseLearning Page — from preparing files to publishing your site live. No servers, no frameworks, no nonsense.

1. Understand What You’re Deploying

HouseLearning Pages hosts static websites. That means:

  • HTML for structure
  • CSS for styling
  • JavaScript for interactivity

There is no backend execution. If it runs in a browser, it works here.

2. Required Files

Every site must include an entry point: 

index.html   ← required
styles.css   ← optional
script.js    ← optional

If index.html is missing, the site will not load.

3. Creating Your Site

From the Pages dashboard:

  1. Click Create New Site
  2. Choose a site name (used in the URL)
  3. The system provisions storage instantly

Site names are URL-safe and permanent.

4. Uploading Files (Best Practices)

Upload your files exactly as they should exist on the site.

  • Use relative paths (./styles.css)
  • Keep filenames lowercase
  • Avoid spaces in filenames
  • Organize images into folders

5. Deployment & What Happens Behind the Scenes

When you deploy:

  • Your files are validated
  • A GitHub Action commits them
  • GitHub Pages publishes the site

Average deployment time is under 30 seconds.

6. Common Setup Mistakes

  • Missing index.html
  • Broken file paths
  • Using server-only features
  • Uploading unsupported file types
