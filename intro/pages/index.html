SafeLibrary Pages Hosting

Hosting SafeLibrary with HouseLearning Pages

HouseLearning Pages lets users publish static sites — including SafeLibrary builds — with zero server configs. It’s a free static hosting layer on top of GitHub Pages that stores site files, deploys them via GitHub Actions, and serves them under `pages.houselearning.org`.

1. What Is HouseLearning Pages?

HouseLearning Pages is a static site platform — users log in, upload HTML/CSS/JS files, and deploy. Your site is automatically published under the `pages.houselearning.org` domain with no backend to manage.

2. How It Works

The hosting workflow is simple:

  • Log in at https://pages.houselearning.org.
  • Create a new site and upload your HTML/CSS/JS files.
  • Deploy — a GitHub Action writes your files into the repository and GitHub Pages serves them.
  • Your site goes live under a predictable URL like pages.houselearning.org/{username}/{siteSlug}/.

3. safelibrary-upload-pgs Repo Overview

The `safelibrary-upload-pgs` project contains the code and workflows that power Pages deployment for SafeLibrary sites. It defines how user site files get committed via a GitHub Action, how Firebase/Firestore storage is structured, and the GitHub Pages hosting pipeline.

Key bits:

  • Hosting platform logic for static sites.
  • GitHub Action to deploy user uploads to Pages.
  • Firestore users/sites/files backend model.

4. Quick Start Example

To host SafeLibrary or any static site:

  1. Log into HouseLearning Pages.
  2. Create a new site project.
  3. Upload HTML/CSS/JS files (e.g., your SafeLibrary build).
  4. Hit “Deploy” and wait for GitHub Pages to serve it live.
